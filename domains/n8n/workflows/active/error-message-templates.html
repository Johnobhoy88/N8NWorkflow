üìò Implementation Guide

How to Use These Templates

  1. Copy the HTML code from the template you need
  2. Replace placeholders (marked with {{ }}) with actual values from your error data
  3. Use in n8n Code nodes or HTML email composition nodes
  4. Test thoroughly with different error scenarios

Placeholder Reference

Placeholder Description Example Value
{{ ERROR_MESSAGE }} User-friendly error description Your email must be valid
{{ ERROR_CODE }} Technical error code INVALID_EMAIL
{{ EXECUTION_ID }} n8n execution identifier abc123xyz789
{{ STAGE }} Pipeline stage where error occurred architect
{{ SEVERITY }} Error severity level HIGH
{{ USER_EMAIL }} User's email address user@example.com

Best Practices

  • Always include: Clear error description, actionable steps, support contact
  • Use appropriate severity: Match visual styling to error severity
  • Be empathetic: Acknowledge user frustration, offer solutions
  • Provide context: Include execution ID and timestamp for debugging
  • Test rendering: Verify templates work across email clients
